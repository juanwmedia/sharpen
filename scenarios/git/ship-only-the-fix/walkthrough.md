The trap is the reflex: git add -A or git commit -am would drag TODO.md (and -A also debug.log) into the fix commit. Stage the one path that belongs to the concern, git add src/invoice.ts, then git commit -m with a message about the fix. git status before committing is the habit that makes atomic commits cheap: it shows exactly what would travel. Any route to the same state passes, for example staging everything and then unstaging the extras with git restore --staged.
