Uncommitted changes travel with you when you switch branches; that is the escape hatch this situation is built on. git switch -c feature/retry-loop creates the branch where HEAD stands and carries the dirty tree along (git checkout -b does the same). Then git commit -am stages every tracked modification and commits in a single move; the deliberate two-step (git add src/client.ts, then git commit -m) is equally valid and passes too. Order is the whole game: committing first would plant the work on main and move the branch you were told to protect. The arena checks main's commit count, so that route fails even though the file content ends up identical.
