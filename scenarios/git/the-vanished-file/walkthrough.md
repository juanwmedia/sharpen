The deletions only happened in the working tree; HEAD still holds every file, and git status lists them as deletions that are not staged. git restore . checks all of them out of HEAD in one move, and it never touches untracked files, so scratch.md survives. Restoring the deleted paths one by one (git restore deploy.sh scripts/healthcheck.sh) lands on the same state and passes too. Sweeping with rm or clean would erase the scratch notes and fail. No commit is needed and none is wanted: nothing was ever staged, so history was never in danger, only the working copy.
