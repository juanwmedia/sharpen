git branch shows the map: you are standing on spike/cache-ttl and main sits one commit behind. Two protections of real git shape the order of moves. First, git refuses to delete the branch you are standing on, so git switch main comes first; the experiment's commit stays behind on its branch and the working tree comes back clean. Then git branch -d spike/cache-ttl hits the second protection: "the branch is not fully merged", because -d only deletes branches whose commits are already reachable from where you stand, and deleting this one throws its commit away. Losing the experiment is exactly the point here, so git branch -D spike/cache-ttl is the deliberate, force spelling that confirms it. git log before and after proves main never moved.
